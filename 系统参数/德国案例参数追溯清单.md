# 德国案例参数追溯清单

**用途**  
这份清单专门用于审计德国案例参数的出处强度，目标不是重复抄录代码，而是回答三个问题：

1. 这个参数能不能在 `系统参数/` 里的正式资料中直接找到？
2. 如果找不到，是不是至少能找到量级一致的参考？
3. 如果两者都不行，它就是当前需要后续补证据的高风险参数。

**判定规则**

- `A 已压实`：能在 `系统参数/` 的 PDF/TXT 中直接找到同一参数或非常接近的明确数值，且可给出页码。
- `B 量级匹配`：找不到完全同值，但能找到用途、技术路线、量级相近的正式资料，可支撑“工程取值合理”。
- `C 暂缺正式来源`：目前只能在代码或历史工作簿中看到，不能由 `系统参数/` 中正式文献/报告直接支撑。

**重要说明**

- 本清单故意**不把代码文件当作来源**。
- 代码里的数值仅用于识别“当前模型实际使用值”。
- 如果一个参数被判为 `C`，不代表它一定错，只代表它目前**还不能作为论文中可直接引用的已证实参数**。

---

## 1. 总体结论

德国案例参数目前可以分成三层：

- `A 已压实`：容量电费、德国工业电/气价量级、燃气机发电效率、燃气机热回收量级、风电/光伏成本量级、电储能自放电量级。
- `B 量级匹配`：电储能效率拆分、热泵成本与 COP 量级、热储能成本量级。
- `C 暂缺正式来源`：德国案例的精确 `ele_price=0.0025 €/kWh`、`ac_cop=0.75`、`ec_cop=2.87`、`ehp_cop=4.44`、热/冷储能精确效率与损失率、整套 `invest_coeff` 精确值。

换句话说：

> 德国案例并不能说“参数基本都已经由 `系统参数/` 里的文献完全压实”。更准确的说法是：关键边界参数与若干主设备参数已有正式资料支撑，但仍有一批核心模型参数属于历史设定或工程取值，需要后续继续清理。

## 3. 风险排序

### 3.1 低风险参数：可以直接写进论文并给出处

- `capacity_charge`
- 德国工业电价/气价现实量级
- `gt_eta_e`
- `gt_eta_h` 的量级说明
- `lambda_e/lambda_h/lambda_c` 的理论定义
- 电储能自放电量级

### 3.2 中风险参数：可保留，但应写成“工程取值/量级一致”

- 风电、光伏年化成本量级
- 电储能年化成本量级
- 热泵 CAPEX/COP 的量级
- 热储年化成本量级
- 热储损失率量级
- 电储能充放电效率拆分

### 3.3 高风险参数：不建议直接说“有文献支撑”

- `ele_price = 0.0025 €/kWh`
- `ac_cop = 0.75`
- `ehp_cop = 4.44`
- `ec_cop = 2.87`
- `hs_cs_charge_eff = 0.90`
- `hs_cs_discharge_eff = 0.90`
- `pcs` 冷储能精确年化系数
- 德国 `invest_coeff` 中所有精确单值

## 4. 建议你后续怎么用

### 4.1 论文正文

正文里优先使用 `A` 类参数，并附“文档 + 页码”。

### 4.2 论文方法部分

对于 `B` 类参数，建议写法：

> 设备参数取值参考本地报告与公开技术成本数据库的量级，并结合本项目历史模型参数表进行工程化设定。

### 4.3 后续清理优先级

最值得继续补证据的德国案例参数是：

1. `ele_price = 0.0025 €/kWh`
2. `ac_cop`
3. `ec_cop`
4. `ehp_cop`
5. 热/冷储效率与损失率
6. 德国 `invest_coeff` 的精确外部出处
