## 1. 文档目的

这份文档是给新进入本仓库的 AI 助手使用的压缩索引入口，目标是：

- 快速建立项目代码结构认知，减少逐文件检索成本

若任务涉及：

- 查看项目结构、理解代码入口、快速建立仓库上下文
- 查找实验目录、结果文件、脚本入口、文档入口
- 跑实验、复现实验、整理结果
- 快速判断应该先看哪些代码文件或文档

优先先读本文件，再决定是否继续打开其他文档或代码。

## 4. 核心代码调用链

主实验调用链：

```text
run.py
  -> cchp_gasolution.py
    -> cchp_gaproblem.py
      -> operation.py
        -> OEMOF / Pyomo / HiGHS / GLPK
```

数据流：

1. `run.py` 解析参数，生成结果目录名，决定是 `--mode` 还是 `--exp`
2. `run.py` 调 `run_comparative_study()` 或批量实验辅助函数
3. `cchp_gasolution.py` 按方法逐个运行：
   - `economic_only`
   - `std`
   - `euclidean`
   - `pearson`
   - `ssr`
4. `cchp_gaproblem.py` 中每个个体会遍历 14 个典型日
5. 每个典型日创建一个 `OperationModel`
6. `operation.py` 调求解器完成调度求解
7. 返回年化经济成本 + 匹配度目标值

### 8.2 后验分析与图表

| 脚本 | 作用 |
|---|---|
| `scripts/post_analysis.py` | 后验运行指标分析（`load_pareto_solutions` 已内置哨兵阈值 `PARETO_INFEASIBLE_THRESHOLD=1e8`，自动过滤 GA 惩罚值） |
| `scripts/post_analysis_report.py` | Phase 2/3 后验报告主入口（从 run.py / run_existing_post_analysis.py 调用） |

| `scripts/phase2_sentinel_filter.py` | **Phase 2 结果二次过滤**（2026-04-18 新增）：扫描批次目录下所有 `Pareto_*.csv` 与 `post_analysis_results.csv`，把 `Economic_Cost/annual_cost > 1e8` 视为不可行解，产出 `*_clean.csv` + `*_infeasible.csv` + `phase2_sentinel_report.md`。支持 `--scan-all` / `--root` / `--threshold` / `--write` / dry-run |

| `scripts/pareto_hypervolume.py` | **Pareto 归一化超体积评估**（2026-04-18 新增）：每场景共享参考点 (cost_max×1.05, matching_max×1.05)，对每 (scenario, exp, method, batch) 算 hv_norm ∈ [0,1]，并按预定义 (no_carnot, with_carnot) 对计算 ΔHV%。支持 `--ref-margin` |
| `scripts/pareto_overlay.py` | **Pareto 前沿叠图**（2026-04-18 新增）：每场景一张图，跨方法跨批次散点叠加 + 非支配前沿折线 + 图例标 HV/cost_min。输出 4 张单场景图 + 1 张 2×2 panel 主图（论文 Figure 直用） |

以下任一变化发生时，必须同步更新本文件：

- 顶层代码结构变化

- `docs/` 目录结构变化

换句话说，凡是会影响下一位 AI 如何快速理解仓库结构的变化，都必须同步更新本文件。
