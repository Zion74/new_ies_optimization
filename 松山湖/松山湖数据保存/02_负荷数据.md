### 4.3 ⚠️ 关于"逐时原始数据"的说明

**图 4 的 8760 个小时数据，PDF 未提供数字表格，仅以图像形式呈现。**

由于 8760 个数据点在图像中几乎逐像素重叠，**无法从图像中读出任意某小时的具体值**。能从图 4 读出来的只有：
- 峰值范围（夏季最高约 500 kW / 3500 kW）
- 季节模式与日周期模式（定性）
- 整体波形包络

如果需要 8760 小时的逐时数据，目前只有三种来源：
1. 向项目方索取原始 csv/xlsx 数据（推荐，但需通过正式渠道）
2. 用图 5 延时曲线（形状）+ 图 3 月度总量（约束）合成（当前做法）
3. 完全人工合成（旧做法，与 PDF 形状不一致，已弃用）

详见 [generate_songshan_data.py](../../scripts/generate_songshan_data.py) 与
`docs/辩论确认/latest_松山湖数据与模型口径辩论共识.md`。

PDF 原图名："社区 A3A4A5 栋 2022 年度电冷负荷延时曲线"

### 5.1 电负荷延时曲线

| 项目 | 数值 |
|------|------|
| 横轴 | 累计小时数 0 ~ 9000 h |
| 纵轴 | 电负荷 0 ~ 600 kW |
| 起点（0 h）峰值 | 约 530 kW |
| 标注关键点 | (2310 h, 321.5 kW) |
| 急剧下降区间 | 约 3000 ~ 3500 h |

物理意义：
- 全年电负荷 > 530 kW 的时间几乎为 0
- 全年电负荷 > 321.5 kW 的时间约 2310 h
- 全年约 5000 h（夜间、周末、节假日）电负荷 < 50 kW

### 5.2 冷负荷延时曲线

| 项目 | 数值 |
|------|------|
| 横轴 | 累计小时数 0 ~ 9000 h |
| 纵轴 | 冷负荷 0 ~ 3600 kW |
| 起点（0 h）峰值 | 约 3460 kW |
| 标注关键点 | (1811 h, 1269 kW) |
| 急剧下降区间 | 约 3000 ~ 3500 h |

物理意义：
- 全年冷负荷 > 3460 kW 的时间几乎为 0（实际同时峰值）
- 全年冷负荷 > 1269 kW 的时间约 1811 h
- 全年约 3500 h 以后无冷负荷需求（冬季）

⚠️ 5797.21 kW（设计冷负荷）与 3460 kW（实际同时峰值）的差异：
设计值是三栋楼非同时峰值之和，实际运行中三栋楼不同时达到峰值，所以同时峰值远低于设计值。优化建模时应使用 3460 kW 而非 5797 kW。

## 7. 模型输入相关换算

```
冷吨与 kW 换算：1 RT = 3.517 kW
所以 82.7 万冷吨 = 82.7 × 10⁴ × 3.517 / 10⁴ = 290.85 万 kW（与 291.15 一致）

平均工作小时电负荷：
166.64 万 kWh / 3300 h ≈ 505 kW（与延时曲线峰值 530 kW 吻合）

平均工作小时冷负荷：
291.15 万 kWh / 2800 h ≈ 1040 kW（远低于峰值 3460 kW，体现峰谷悬殊）
```
