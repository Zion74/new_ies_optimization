## 图5 "电冷负荷延时曲线" 是什么意思

延时曲线（Load Duration Curve）是把全年8760小时的负荷数据**从大到小排序**后画出来的曲线，横轴是小时数（0~8760h），纵轴是负荷功率（kW）。

它的意义是：**告诉你"有多少小时负荷超过某个值"**。

举个例子，如果冷负荷延时曲线在 x=2000h 处对应 y=2500kW，意思是全年有2000个小时冷负荷超过2500kW。

对于松山湖项目，这张图直接决定了装机容量的选择：

- 曲线最左端（x=0）= 峰值负荷 5797kW → 但只出现极少时间
- PDF提到"全年冷负荷量占比最高的负荷段为2000~2500kW" → 这就是延时曲线中间最"厚"的部分
- 设计装机时不需要覆盖峰值，用蓄冷来"削峰"，所以选了1000RT蓄冷

## 4.3 项目装机 — "装多大"

这节回答的问题是：**每种设备装多大容量？依据是什么？**

PDF给出了4个设计要点：

**要点1：运行时间约束**

> 负荷需求时间8:00-19:00，系统供能不超3300小时；供冷季4-11月，不超2800小时

意思是这个园区白天办公才有负荷，不是24小时工厂。全年真正需要供能的时间有限。

**要点2：年总量上限**

> 年供电不超84万kWh；供冷不超291万kWh

这是从图3/图4的历史数据算出来的总需求量。

**要点3：负荷分布特征（这就是延时曲线告诉我们的）**

> 全年电负荷量占比最高的负荷段为400~~500 kW 全年冷负荷量占比最高的负荷段为2000~~2500 kW

这句话的意思是：虽然峰值冷负荷有5797kW，但大部分时间冷负荷集中在2000~2500kW这个区间。所以设备容量不需要按5797kW来配，而是按"最常出现的负荷段"来配基荷设备，超出的部分用蓄冷来补。

这就是为什么最终装机是：

- 内燃机 500kW（对应电负荷最高频段400~500kW）
- 溴化锂 550kW（用内燃机余热驱动制冷）
- 电制冷 1500kW（补充制冷）
- 蓄冷 1000RT（削峰，覆盖超出部分和夜间蓄冷）

**要点4：蓄冷时段**

> 夜间低谷蓄冷时间0:00-8:00，每天蓄冷8小时

利用低谷电价（0.34元）蓄冷，白天释放，省钱。

## 4.4 运行策略及供需匹配 — "怎么运行"

这节回答的问题是：**一年12个月，每个月设备怎么开？谁先谁后？**

**核心策略："以冷定电"**

这四个字是关键。意思是：不是根据电负荷来决定内燃机开多大，而是根据冷负荷来决定。因为松山湖是冷主导系统（冷电比4.18），冷负荷远大于电负荷，所以制冷需求决定了整个系统的运行方式。

**具体运行安排：**

| 月份        | 内燃机状态 | 负荷率 | 原因                                                         |
| ----------- | ---------- | ------ | ------------------------------------------------------------ |
| 1-4月、12月 | **停机**   | 0%     | 冬春季冷负荷很小（2~5万kW/月），不值得开内燃机，用蓄冷+电网就够了 |
| 5月         | 变工况     | 56%    | 初夏，冷负荷开始上升但还不大                                 |
| 6月         | 变工况     | 78%    | 冷负荷继续上升                                               |
| 7-10月      | **满负荷** | 100%   | 盛夏，冷负荷最大（40~49万kW/月），内燃机全开                 |
| 11月        | 变工况     | 67%    | 秋末，冷负荷下降                                             |

**供需匹配的优先级顺序：**

PDF第20-25页那些逐月的图表展示的就是每个月的能量流向：



```
天然气 → 内燃机 → 发电（供用户电 + 驱动电制冷）
                 → 余热 → 溴化锂制冷
                 
光伏 → 发电（自用 or 上网）

电网 → 购电（补充不足 + 低谷蓄冷）

蓄冷 → 夜间蓄冷，白天释放
```

以7月（满负荷）为例，从表4可以看到能量流向：

- 内燃机耗气 2.79万Nm³ → 发电 11.88万kWh
- 发电中：3.13万kWh供用户电，8.51万kWh驱动电制冷
- 余热驱动溴化锂制冷 12.47万kW
- 电制冷产冷 34.05万kW
- 蓄冷释放 2.82万kW
- 合计供冷 = 12.47 + 34.05 + 2.82 = 49.34万kW ✓

而1-4月内燃机停机时：

- 光伏发电供用户电
- 电网购电补充
- 蓄冷（用低谷电蓄冷）满足少量冷需求

**这对我们的优化模型意味着什么：**

我们的 `operation.py` 中的 OEMOF 模型会自动做类似的调度决策——求解器会根据电价、设备效率、负荷需求自动决定每小时每台设备开多少。松山湖PDF中的运行策略相当于人工版的"最优调度"，而我们的模型用数学优化来自动找到这个策略。

这些逐月能量流向图对你的容量优化有两个层面的价值：

## 直接价值：验证你的模型是否合理

这些图本质上是松山湖项目**人工设计的最优调度方案**。你的 OEMOF 模型跑出来的结果，应该和这些图呈现的能量流向趋势一致。比如：

- 7月满负荷时，内燃机发电大部分用于驱动电制冷（8.51万kWh），只有少部分直接供用户电（3.13万kWh）→ 如果你的模型跑出来7月内燃机发的电全供用户、不制冷，那说明模型有问题
- 1-4月内燃机停机，全靠光伏+电网+蓄冷 → 如果你的模型在冬季还大量开内燃机，也不对

所以这些图是你**调试和验证模型的参照物**。

## 间接价值：其实不大

说实话，对容量优化本身帮助有限。原因是：

这些图展示的是**给定装机容量后的运行调度结果**（500kW内燃机+500kW光伏+1000RT蓄冷已经确定了）。而你做的是**反过来的问题**——决策变量就是这些设备的容量，让优化算法自己找最优配置。

你的优化框架是两层结构：

- 上层（遗传算法）：搜索设备容量组合
- 下层（OEMOF）：给定容量后，自动求解最优调度

下层的OEMOF会自动算出类似PDF这些图的能量流向，不需要你手动指定"几月开机几月停机"。所以这些图的运行策略信息已经隐含在你的模型里了。

**真正对容量优化有用的是表4的月度总量数据**（我们已经用了）和**延时曲线的形状特征**（决定了合理的容量搜索范围），而不是这些逐月能量流向图本身。
