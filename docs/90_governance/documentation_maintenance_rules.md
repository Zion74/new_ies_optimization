# 文档维护规则

更新时间：2026-04-10

## 1. 适用范围

凡是涉及以下内容，默认都要同步维护 `docs/`：

- 会议论文、SCI 论文、硕士论文的逻辑结构变化
- 研究主线、章节边界、创新点边界变化
- 核心实验链、关键图表清单、代码映射变化
- 文档迁移、合并、归档与目录清理

## 2. 默认工作流

每次更新论文逻辑结构时，默认执行以下动作：

1. 先判断影响的是会议、SCI、硕士论文哪一层。
2. 更新对应层级的 `latest_logic_structure.md`。
3. 更新对应层级的实验 / 图表 / 代码映射文档。
4. 如果三层关系也变了，再同步更新：
   - `docs/01_overview/latest_research_architecture.md`
   - `docs/01_overview/three_layer_mapping.md`
5. 如果旧文档被替代，移动到 `docs/99_archive/` 并留下最小说明。

## 3. 稿件源码目录纪律

- `论文撰写/会议/`：只保留正式源码、数据、图表、README、编译产物。
- `论文撰写/paper/`：只保留正式源码、图表、参考文献、README、编译产物。
- 支持材料进入 `论文撰写/support/`。
- 汇报材料进入 `论文撰写/reports/`。
- 被替代的旧结构稿、旧会议归档、旧思路笔记进入 `docs/99_archive/`。

## 4. 触发词提醒

当用户提到以下任一语境时，不要直接在散落旧笔记里改：

- 会议论文 / IEEE / conference paper 主线
- SCI / journal paper / Applied Energy 主线
- 硕士论文 / thesis / chapter architecture
- 研究架构、三层映射、实验-图表-代码映射
- 文档整理、文档沉淀、合并笔记、归档旧文档
- “以后每次更新论文逻辑结构时，也同步更新文档”

正确动作是：先读 `skills/research-doc-maintainer/SKILL.md` 和 `docs/README.md`，再按权威文档体系更新。

## 5. 交付检查清单

提交这类任务前至少检查：

- 是否只保留一份活跃的最新结构说明。
- 是否已更新对应层级的逻辑结构文档。
- 是否已更新实验 / 图表 / 代码映射。
- 是否已把被替代的旧笔记归档。
- 是否已把源码目录中的支持性材料迁出。
