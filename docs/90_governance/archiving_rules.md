# 归档规则

更新时间：2026-04-10

## 1. 哪些内容必须归档

出现以下情况时，不应继续留在活跃目录：

- 被新的 `latest_logic_structure.md` 替代的旧结构稿
- 散落在 `研究思路/`、`论文撰写/` 根目录的旧研究主线笔记
- 已不再属于正式稿件源码目录的会议草稿、旧会议归档、旧审查备忘
- 与当前主线重复、但又不适合直接删除的历史说明文档

## 2. 归档路径规范

统一放到：

- `docs/99_archive/legacy_notes/YYYY-MM-DD_<topic>/...`

推荐做法：

- 同一轮整理任务归到同一个日期文件夹
- 目录名尽量直接说明来源，例如 `研究思路/`、`论文撰写_root/`、`会议_pre_iemi_archive/`

## 3. 归档时的最小动作

1. 移动旧文件，不保留活跃目录下的重复副本。
2. 在归档根目录写一个 README，说明为什么移动、现在应该看哪里。
3. 如果源目录会让人误解，补一个 README 指向新的权威入口。
4. 更新相关 README，避免旧路径继续被当作“最新说明”。

## 4. 哪些内容不要随便归档

- 正在使用的 LaTeX 主稿与章节源码
- 活跃实验数据与正式图表
- 当前仍被脚本直接消费的 CSV / 结果文件
- 当前权威的 `docs/` 结构文档

## 5. 删除原则

- 对“历史上可能还有参考价值”的文件，优先归档，不直接删除。
- 只有明显空文件、重复产物且无历史价值时，才考虑直接删除。
