 传统的源荷匹配度通常是把“电”和“热”分开算的（比如分别计算电的波动率和热的波动率，然后简单加权），这实际上忽略了综合能源系统最本质的特征——**“以热定电”或“以电定热”的强耦合约束**。

在分布式“电-热”系统中，核心设备燃气轮机（GT）产生的电和热是**同时产生、比例固定（或在一定范围内调节）**的。当你为了匹配电负荷而调整GT出力时，必然会改变热出力，可能导致热侧的源荷不匹配。

# 方案1 基于“能质系数”加权的耦合源荷不匹配度

为了量化这种耦合关系，我建议你引入**“能质系数（Energy Quality Coefficient）”**的概念。

理论依据：

电能是高品位能源（纯佣），热能是低品位能源（佣+无用能）。从热力学第二定律来看，1kWh的电能不匹配，其代价远高于1kWh的热能不匹配。同时，电网对平衡的要求是“秒级”的，而热网具有“热惯性”，对不匹配的容忍度更高。

**新指标定义：综合加权欧氏距离匹配度 (Weighted Euclidean Distance Matching Degree)**

我们不再简单的线性相加，而是将某一时刻的“电净负荷”和“热净负荷”看作一个**二维向量**，计算这个向量到原点（完美匹配点 (0,0)）的加权距离。

**公式设计：**

$$F_{coupled} = \sum_{t=1}^{T} \sqrt{ \left( \lambda_e(t) \cdot P_{net,e}(t) \right)^2 + \left( \lambda_h \cdot P_{net,h}(t) \right)^2 }$$

- $P_{net,e}(t)$：$t$时刻的电能净负荷（$|P_{grid} - P_{overflow}|$，即与电网的交互量）。
- $P_{net,h}(t)$：$t$时刻的热能净负荷（热源出力 - 热负荷，或者热储能的非计划动作量）。**注：在你的代码逻辑里，如果没有热网交互，热净负荷可以定义为“原本应该由热储能填补的缺口”或者“废弃的热量”。**
- $\lambda_e(t)$：**电能权重（分时电价权重）**。如前所述，高峰电价时，电的不匹配惩罚极大。
- $\lambda_h$：**热能权重（能质系数）**。通常取 0.3~0.5（基于卡诺循环效率或工程经验），表示热的价值低于电。或者如果强调供热保障，可以设为 1.0。

**这个指标的物理意义（创新点）：**

1. **耦合惩罚（平方项）：** 使用欧氏距离（平方开根号）意味着，如果系统在同一时刻“电”和“热”都出现了偏差，指标会呈现非线性的增长。这迫使优化算法去寻找那些能**同时平衡**电热的运行点，而不是拆东墙补西墙。
2. **能质差异：** 通过 $\lambda_e$ 和 $\lambda_h$ 的差异，体现了“电比热贵/难平衡”的物理事实。

# 方案2 基于“㶲平衡”的源荷匹配度 (Exergy-based Source-Load Matching)

#### 1. 核心理论与创新点

这种方法的创新逻辑在于**“从数量匹配升级到质量匹配”**。

- **痛点**：传统的“能量（Energy）”视角认为 $1\text{kWh}$ 的电和 $1\text{kWh}$ 的热在数值上是等价的，直接相加或加权往往缺乏物理依据。
- **创新点**：引入热力学第二定律中的**“㶲（Exergy）”**概念。
  - **电的㶲值**：电是纯㶲，能质系数为 1。
  - **热的㶲值**：热是低品位能源，其“有效能”取决于温度。温度越高，价值越高。
  - **统一标尺**：将电和热都折算成“㶲”进行叠加，消除了不同能源形式之间的“品位壁垒”，使得匹配度的定义在物理上更加严谨。

#### 2. 数学公式

目标是最小化**系统总㶲供需偏差**：

$$F_{exergy} = \sum_{t=1}^{T} \left| Ex_{net}(t) \right| = \sum_{t=1}^{T} \left| Ex_{net, e}(t) + Ex_{net, h}(t) \right|$$

其中：

- **电㶲净负荷**：$Ex_{net, e}(t) = 1 \cdot P_{net, e}(t)$ （电的能质系数为1）
- **热㶲净负荷**：$Ex_{net, h}(t) = \left( 1 - \frac{T_0}{T_h} \right) \cdot P_{net, h}(t)$
  - $T_0$：环境参考温度（通常取 298.15K 或 25℃）。
  - $T_h$：供热温度（开尔文）。例如，如果你的系统供应的是 80℃ 的生活热水，$T_h = 353.15K$。
  - **卡诺因子 $\eta_c = 1 - T_0/T_h$**：这就是热的“能质系数”。

------

### 3. 代码实现指南

你需要修改 `gaproblem.py` 中的 `subAimFunc`。

#### 修改逻辑：

1. **引入温度参数**：设置环境温度 $T_0$ 和供热温度 $T_h$。
2. **计算卡诺因子**：计算热转为㶲的比例。
3. **统一计算**：将电和热的净交互量折算为㶲后相加，取绝对值累积。

Python

```
#

def subAimFunc(args):
    # ... (参数解包和变量定义保持不变) ...
    
    # 1. 定义热力学参数 (Exergy Calculation Parameters)
    # 环境参考温度 (Kelvin), 假设为 25℃
    T0 = 25 + 273.15 
    # 供热/用热温度 (Kelvin), 假设分布式供暖/热水温度为 70℃
    # 如果是蒸汽，温度会更高；如果是地暖，温度会更低。这个参数对结果影响很大，是论文可以讨论的点。
    Th = 70 + 273.15 
    
    # 计算热的“能质系数” (卡诺因子)
    # 物理意义：1单位的热能中，只有这么大比例是“高价值”的，其余是无法做功的
    carnot_factor = 1 - T0 / Th
    
    # 初始化㶲匹配度指标
    exergy_mismatch_score = 0
    
    # ... (OperationModel 初始化和运行保持不变，保留电、热、气相关，去除冷相关) ...
    
    # ... (获取结果 complementary_results 后) ...

    for d in typical_days[cluster_medoid]:
        # start_index = (d - 1) * 24 
        for k in range(24):
            # --- A. 计算电不匹配量 (本身就是㶲) ---
            # 物理含义：与电网的交互量 (买电 - 卖电)
            # 注意：这里我们取有符号数值，因为“缺电”和“余电”在能量守恒里是相反的
            # 但在匹配度评估中，我们关注的是绝对偏差的总量
            
            p_net_ele = complementary_results["grid"][k] - complementary_results["electricity overflow"][k]
            ex_net_ele = p_net_ele * 1.0  # 电的能质系数为 1
            
            # --- B. 计算热不匹配量 (折算为㶲) ---
            # 物理含义：热源出力 + 溢出热量 (即系统为了平衡而多产出或少产出的热)
            # 这里假设 heat_source 包含了不足时补充的热，heat_overflow 是多余扔掉的热
            p_net_heat = complementary_results["heat source"][k] + complementary_results["heat overflow"][k]
            ex_net_heat = p_net_heat * carnot_factor # 热折算为㶲
            
            # --- C. 计算总㶲不匹配度 ---
            # 创新点：我们在“㶲”的层面上进行加和
            # 这意味着：1kW的电不匹配，比1kW的热不匹配，惩罚要重得多（大约重5-8倍，取决于温度）
            # 这天然地迫使系统优先保障电的平衡，这符合“以电定热”或电网调峰的实际需求
            
            exergy_mismatch_score += abs(ex_net_ele) + abs(ex_net_heat)

    # ... (计算经济目标保持不变) ...

    return [economic_obj_i, exergy_mismatch_score]
```

# 方案三：基于“供需重叠度”的综合匹配指标 (Supply-Demand Overlap Degree)

#### 1. 核心理论与创新点

这种方法回归了“源荷匹配”的最本质定义：**系统自给自足的能力**。

- **物理意义**：想象“源出力曲线”和“荷需求曲线”围成的面积。两者的**重叠部分**就是系统“自发自用”的能量，**不重叠部分**（要么是缺电需网购，要么是余电需上网）就是不匹配量。
- **创新点（耦合视角）**：
  - 将电和热的重叠度（Load Satisfaction Rate, LSR）进行加权耦合。
  - **不同于波动率**：波动率低不代表匹配好（比如一直缺电，波动也很小）。重叠度直接反映了对外部电网/热网的**零依赖程度**。

#### 2. 数学公式

目标是**最大化**重叠度，即**最小化**（1 - 重叠度）。

$$F_{overlap} = \lambda_e \cdot (1 - \text{LSR}_e) + \lambda_h \cdot (1 - \text{LSR}_h)$$

其中，某一能流 $i$（电或热）的负荷满足率（重叠度）定义为：



$$\text{LSR}_i = \frac{\sum_{t=1}^{T} \min(P_{gen, i}(t), P_{load, i}(t))}{\sum_{t=1}^{T} P_{load, i}(t)}$$

- $P_{gen, i}(t)$：系统内部供应量（燃气轮机+光伏+风电+储能放能）。
- $P_{load, i}(t)$：用户需求量（基础负荷+电制冷/热泵消耗）。

#### 3. 代码实现

修改 `gaproblem.py` 中的 `subAimFunc`：

```
            # 电不匹配量 = 买电量 + 卖电/弃电量
            unmatched_ele = complementary_results["grid"][k] + complementary_results["electricity overflow"][k]
            
            # 热不匹配量 = 补热量(Heat Source) + 弃热量(Heat Overflow)
            unmatched_heat = complementary_results["heat source"][k] + complementary_results["heat overflow"][k]
            
            sum_overlap_ele += unmatched_ele
            sum_overlap_heat += unmatched_heat
            
            # 注意：如果要把这个做成 "率" (0-1)，需要除以总负荷。
            # 如果只是为了优化，直接最小化 "总不匹配能量绝对值" 也是等效的。
            # 这里为了简单，我们直接返回加权的 "总不匹配能量 (kWh)"
            
    # 计算最终指标：加权总不匹配量
    # 物理含义：规划期内，没能“自发自用”的能量总和（包括不得不买的和不得不扔的）
    overlap_mismatch_score = w_e * sum_overlap_ele + w_h * sum_overlap_heat

    return [economic_obj_i, overlap_mismatch_score]
```

# 方案四：基于“灰色关联度”的匹配指标 (Grey Relational Analysis, GRA)

#### 1. 核心理论与创新点

这是一种在系统工程和中文核心期刊中非常受青睐的方法（看起来数学很高级）。

- **核心思想**：它不只看数值大小，还看**“曲线形状的相似程度”**。如果源出力曲线和负荷曲线的**变化趋势**高度一致（同增同减），说明调节性能极好，即便数值上有缺口，也容易通过储能平抑。
- **创新点**：将“多能流”构建成一个多维序列，计算**源-荷序列的灰色关联度**。

#### 2. 数学公式

目标是**最大化**关联度 $\gamma$，即**最小化** $(1 - \gamma)$。

$$\xi(t) = \frac{\min \min \Delta(t) + \rho \cdot \max \max \Delta(t)}{\Delta(t) + \rho \cdot \max \max \Delta(t)}$$

- $\Delta(t) = |X_{load}(t) - X_{gen}(t)|$ ：$t$时刻的绝对差值（需归一化）。
- $\rho$：分辨系数，通常取 0.5。
- $\gamma = \frac{1}{T} \sum \xi(t)$：最终关联度。

#### 3. 代码实现

这个方法计算稍微复杂一点，需要先对数据做**归一化（Normalization）**处理。

Python

```
def subAimFunc(args):
    # ... (参数解包) ...
    
    # 容器，用于存储全年的序列，算整体关联度
    list_load_e, list_gen_e = [], []
    list_load_h, list_gen_h = [], []
    
    # ... (模型运行) ...
    
    # ... (在循环中收集数据) ...
    for d in typical_days[cluster_medoid]:
        for k in range(24):
            # 获取电负荷与电发电 (估算)
            # 假设 load ≈ 原始load, gen ≈ load - grid + overflow
            # 为了简化，我们直接比较 "电网交互曲线" 和 "0基准线" 的距离
            # GRA通常比较两个序列的相似性。
            # 这里我们定义：理想序列 X0 = [0, 0, ..., 0] (完全无交互)
            # 实际序列 Xi = [Net_Ele(t), Net_Heat(t)]
            
            net_ele = abs(complementary_results["grid"][k] - complementary_results["electricity overflow"][k])
            net_heat = abs(complementary_results["heat source"][k] + complementary_results["heat overflow"][k])
            
            list_gen_e.append(net_ele) # 这里存的是偏差量
            list_gen_h.append(net_heat)
            
    # --- 计算灰色关联度 (简化版：针对“零偏差”目标的关联) ---
    # 1. 归一化 (防止电热数量级不同)
    arr_e = np.array(list_gen_e)
    arr_h = np.array(list_gen_h)
    
    # 避免除以0
    norm_e = arr_e / (np.max(arr_e) + 1e-5)
    norm_h = arr_h / (np.max(arr_h) + 1e-5)
    
    # 2. 计算关联系数 (Target is 0)
    # Delta = |Norm_Value - 0| = Norm_Value
    rho = 0.5
    min_diff = 0 # 理论最小差是0
    max_diff = 1 # 归一化后最大差是1
    
    xi_e = (min_diff + rho * max_diff) / (norm_e + rho * max_diff)
    xi_h = (min_diff + rho * max_diff) / (norm_h + rho * max_diff)
    
    # 3. 计算关联度 (平均值)
    gra_e = np.mean(xi_e)
    gra_h = np.mean(xi_h)
    
    # 4. 综合评价 (加权)
    # GRA越大越好 (接近1表示接近0偏差)，所以目标是最小化 (1 - GRA)
    total_gra_score = 0.6 * (1 - gra_e) + 0.4 * (1 - gra_h)
    
    return [economic_obj_i, total_gra_score]
```

### 四种方法的优劣势对比与选择建议

| **方法**                        | **定义核心**      | **优势 (Pros)**                                          | **劣势 (Cons)**                                              | **推荐指数**                                                 | **适用场景**                           |
| ------------------------------- | ----------------- | -------------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | -------------------------------------- |
| **1. 标准差/波动率** (师兄原版) | $\sigma(P_{net})$ | 代码最简单，计算快。                                     | 物理意义弱，无法区分“多电”还是“缺电”，无法耦合电热。         | ⭐⭐                                                           | 单一储能优化                           |
| **2. 㶲平衡法** (Exergy)        | $\sum             | Ex_{net}                                                 | $                                                            | **理论逼格最高**，完美解决电热异质问题，SCI/EI最爱。         | 需要假设温度参数，解释起来稍显晦涩。   |
| **3. 供需重叠度** (Overlap)     | $\sum             | P_{unmatched}                                            | $                                                            | **物理意义最直观**，解释简单（“自给自足率”），工程实用性强。 | 数学形式较简单，可能被质疑创新度不够。 |
| **4. 灰色关联度** (GRA)         | Shape Similarity  | **数学形式最复杂**，看起来很“智能”，适合处理非线性关系。 | 计算繁琐，归一化过程容易丢失绝对数值信息（比如偏差很大但形状很好）。 | ⭐⭐⭐                                                          | **算法研究型**                         |

### 最终建议

**我依然最推荐“方案二：基于㶲（Exergy）的匹配度”或者“加权欧氏距离”**。

- **理由**：对于EI会议论文，你需要一个**看起来有理论深度**（Exergy/Euclidean）但**实现起来不至于把自己坑进去**（GRA需要复杂的归一化）的方法。
- **如果不喜欢“㶲”的概念**，**“加权供需重叠度（方案三）”是第二好的选择，因为它实际上就是“加权欧氏距离”的变体（使用的是曼哈顿距离 L1-norm），你可以给它起个好听的名字，叫“基于能质加权的供需协同指数 (Energy-Quality Weighted Supply-Demand Synergy Index)”**，听起来就很高大上了。
