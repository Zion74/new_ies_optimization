# 第4章：基于负荷预测的日前优化调度实验方案

## 一、实验设计总览

### 核心问题

规划方案确定后（设备容量固定），在日前调度阶段面临负荷和可再生出力的预测不确定性。
本章回答：**㶲加权规划方案在面对预测误差时，是否比等权重方案更鲁棒？**

### 实验逻辑

```
第2/3章的规划结果（设备容量固定）
    ↓
日前调度（用预测负荷+气象预报制定调度计划）
    ↓
实时评估（用实际负荷评估调度计划的执行效果）
    ↓
对比：EQD方案 vs Std方案的性能退化差异
```

---

## 二、数据划分方案

### 方案：四季代表月测试

```
训练集（8个月）：Feb, Mar, May, Jun, Aug, Sep, Nov, Dec
测试集（4个月）：Jan(冬), Apr(春), Jul(夏), Oct(秋)
```

**文献支撑**：

- 时序数据的训练-测试划分必须保持时间顺序，避免未来信息泄露。
对于需要覆盖全年季节性的场景，"留出代表月"（leave-representative-months-out）
是一种常见做法，确保测试集覆盖所有季节工况。
- 参考：Bergmeir & Benítez (2012), "On the use of cross-validation for time series
predictor evaluation", Information Sciences, 191, 192-213.
该文系统讨论了时序交叉验证方法，指出对于有季节性的数据，
需要确保验证集覆盖不同季节。
- 参考：Hong et al. (2016), "Probabilistic energy forecasting: Global Energy
Forecasting Competition 2014 and beyond", International Journal of Forecasting,
32(3), 896-913. GEFCom2014竞赛采用了类似的分季节评估策略。

**具体操作**：

- 每个测试月的前48小时（模型输入窗口）使用该月之前的真实历史数据
- 训练时使用8个月的全部数据（约5840小时）
- 测试时逐日滚动预测：用过去48小时真实值预测未来24小时

**季节性分析**：

- 分季节报告预测精度（冬/春/夏/秋）
- 重点关注：冬季热负荷预测 vs 夏季冷负荷预测的精度差异

## 三、预测模型设计

### 3.1 模型架构：多任务学习（共享Encoder + 多头Decoder）

```
输入（过去48小时 × 8个特征）：
  - ele_load[t-48:t]        # 历史电负荷
  - heat_load[t-48:t]       # 历史热负荷
  - cool_load[t-48:t]       # 历史冷负荷
  - temperature[t-48:t]     # 历史温度
  - solarRadiation[t-48:t]  # 历史太阳辐射
  - windSpeed[t-48:t]       # 历史风速
  - hour_sin[t-48:t]        # sin(2π·hour/24)
  - hour_cos[t-48:t]        # cos(2π·hour/24)

    ↓
共享 Encoder（GRU / LSTM / Transformer）
    ↓
  ┌─────────┬─────────┬─────────┐
  ↓         ↓         ↓
Decoder_e  Decoder_h  Decoder_c
  ↓         ↓         ↓
ele[t:t+24] heat[t:t+24] cool[t:t+24]
```

**文献支撑**：

- 多任务学习用于多能负荷联合预测已有成熟文献：
Xuan et al. (2021), "A multi-energy load prediction model based on deep
multi-task learning and ensemble approach for regional integrated energy
systems", International Journal of Electrical Power & Energy Systems, 126,
  1. doi:10.1016/j.ijepes.2020.106583
  该文使用共享底层+多任务输出的架构同时预测电热冷负荷，
  证明多任务学习能利用负荷间的耦合关系提升预测精度。
- Wang et al. (2023), "Multi-energy load forecasting for regional integrated
energy systems considering temporal dynamic and coupling characteristics",
Energy, 282, 128198. doi:10.1016/j.energy.2023.128198
该文提出考虑时间动态和耦合特性的多能负荷预测方法。

### 3.2 时间特征编码

使用正弦/余弦编码而非one-hot编码：

```python
hour_sin = sin(2π × hour / 24)
hour_cos = cos(2π × hour / 24)
```

**文献支撑**：

- 正弦编码是时序预测中处理周期性特征的标准做法。
参考：Kazemi et al. (2019), "Time2Vec: Learning a General-Purpose
Representation of Time", arXiv:1907.05321.

### 3.3 训练细节

- 损失函数：MSE（三个输出的加权和，权重均为1/3）
- 优化器：Adam, lr=1e-3, weight_decay=1e-5
- 早停：验证集loss连续20个epoch不下降则停止
- 验证集：从训练集中取最后一个月（Dec）作为验证集
- Batch size：64
- 输入窗口：48小时
- 输出窗口：24小时

### 3.4 对比基线


| 模型          | 描述                    | 角色      |
| ----------- | --------------------- | ------- |
| Persistence | 用昨天同时刻的值作为预测          | naive基线 |
| 单任务GRU      | 三个独立的GRU模型分别预测        | 消融对比    |
| 多任务GRU（本文）  | 共享Encoder + 3个Decoder | 主模型     |


**注意**：预测模型对比只是辅助内容（2-3页），重点是调度实验。

### 3.5 预测评价指标


| 指标    | 公式                       | 用途              |
| ----- | ------------------------ | --------------- |
| MAE   | mean(|y - ŷ|)            | 绝对误差            |
| RMSE  | sqrt(mean((y - ŷ)²))     | 对大误差敏感          |
| MAPE  | mean(|y - ŷ| / y) × 100% | 相对误差（注意零值处理）    |
| R²    | 1 - SS_res/SS_tot        | 拟合优度            |
| nRMSE | RMSE / mean(y) × 100%    | 归一化RMSE，便于跨负荷对比 |

## 四、可再生能源预测

### 4.1 方案：物理模型 + 气象预报噪声

**光伏出力预测**：

```python
PV_output = solar_radiation_forecast × PV_capacity × η_pv
```

其中 η_pv 为光伏系统效率（含逆变器损耗，约0.15-0.18）。

**风电出力预测**：

```python
# 简化的风电功率曲线
if wind_speed < v_cut_in:
    WT_output = 0
elif wind_speed < v_rated:
    WT_output = WT_capacity × (wind_speed / v_rated) ** 3
else:
    WT_output = WT_capacity
```

### 4.2 气象预报误差模拟

**研究思路\毕业论文**

**误差参数设置及文献支撑**：


| 气象变量 | 误差模型   | 参数        | 文献依据   |
| ---- | ------ | --------- | ------ |
| 太阳辐射 | 乘性高斯噪声 | σ = 15%   | [1][2] |
| 风速   | 乘性高斯噪声 | σ = 10%   | [3]    |
| 温度   | 加性高斯噪声 | σ = 1.5°C | [4]    |


**文献**：

[1] Antonanzas et al. (2016), "Review of photovoltaic power forecasting",
    Solar Energy, 136, 78-111. doi:10.1016/j.solener.2016.06.069
    该综述报告日前光伏功率预测的nRMSE典型范围为10-30%，
    取决于气候条件和预测方法。NWP模型的日前辐射预测nRMSE
    通常在15-25%范围内。

[2] Yang et al. (2022), "The value of solar forecasts and the cost of their
    errors: A review", Renewable and Sustainable Energy Reviews, 189, 113915.
    doi:10.1016/j.rser.2023.113915
    该RSER综述系统量化了太阳能预测误差对运行成本的影响，
    报告日前预测的nRMSE在15-30%范围内。

[3] Foley et al. (2012), "Current methods and advances in forecasting of wind
    power generation", Renewable Energy, 37(1), 1-8.
    doi:10.1016/j.renene.2011.05.033
    该文报告日前风电功率预测的nRMSE通常在10-20%范围内。

[4] 温度预报误差1-2°C是气象学界的共识。
    参考：Hamill & Whitaker (2006), "Probabilistic quantitative precipitation
    forecasts based on reforecast analogs", Monthly Weather Review, 134(11),
    3209-3229. 日前温度预报的MAE通常在1-2°C。

**实现代码**：

```python
import numpy as np

def simulate_weather_forecast(actual_data, seed=42):
    """模拟日前气象预报数据"""
    rng = np.random.RandomState(seed)

    forecast = {}

    # 太阳辐射：乘性噪声，nRMSE ≈ 15%
    noise_solar = 1 + rng.normal(0, 0.15, len(actual_data['solarRadiation']))
    forecast['solarRadiation'] = np.clip(
        actual_data['solarRadiation'] * noise_solar, 0, None
    )

    # 风速：乘性噪声，nRMSE ≈ 10%
    noise_wind = 1 + rng.normal(0, 0.10, len(actual_data['windSpeed']))
    forecast['windSpeed'] = np.clip(
        actual_data['windSpeed'] * noise_wind, 0, None
    )

    # 温度：加性噪声，MAE ≈ 1.5°C
    noise_temp = rng.normal(0, 1.5, len(actual_data['temperature']))
    forecast['temperature'] = actual_data['temperature'] + noise_temp

    return forecast
```

## 五、调度实验设计

### 5.1 实验前提

从第2/3章中选取规划方案（德国案例）：


| 方案       | 来源                 | 设备容量 |
| -------- | ------------------ | ---- |
| EQD方案    | 第2章Pareto前沿+50%预算点 | 固定   |
| Std方案    | 第2章相同成本水平的方案       | 固定   |
| EQD+CB方案 | 第3章有卡诺电池的方案        | 固定   |


### 5.2 调度场景


| 编号  | 负荷信息 | 可再生出力信息 | 含义         |
| --- | ---- | ------- | ---------- |
| S1  | 实际值  | 实际值     | 完美信息（理论上界） |
| S2  | 预测值  | 气象预报值   | 现实可实现的日前调度 |


**文献支撑**：

- "完美信息 vs 预测信息"的对比是量化预测价值的标准方法。
参考：Morales et al. (2014), "Integrating Renewables in Electricity Markets:
Operational Problems", Springer. 该书系统阐述了"value of perfect information"
的概念和计算方法。
- 两阶段调度（日前+实时）是微电网/IES调度的标准框架。
参考：Parisio et al. (2014), "A model predictive control approach to
microgrid operation optimization", IEEE Transactions on Control Systems
Technology, 22(5), 1813-1827. doi:10.1109/TCST.2013.2295737

### 5.3 两阶段调度流程

**Stage 1：日前调度（Day-ahead）**

```
输入：
  - 预测负荷（模型输出）：ele_load_pred, heat_load_pred, cool_load_pred
  - 气象预报（加噪声）：solar_forecast, wind_forecast
  - 设备容量（固定）：来自规划结果
  - 能源价格：24小时电价、气价

求解：
  - 调用 OperationModel（oemof）
  - 目标：最小化日运行成本
  - 输出：24小时调度计划
    - 各设备出力计划：GT_plan, HP_plan, EC_plan, AC_plan
    - 储能策略：ES_charge_plan, ES_discharge_plan, HS/CS同理
    - 购电计划：grid_plan
```

**Stage 2：实时评估（Real-time evaluation）**

```
输入：
  - 实际负荷：ele_load_actual, heat_load_actual, cool_load_actual
  - 实际气象：solar_actual, wind_actual
  - 日前调度计划（固定）：GT_plan, HP_plan, ...

评估逻辑：
  - CHP/热泵/制冷机出力按日前计划执行（不可调整）
  - 储能充放电按日前计划执行（不可调整）
  - 电网购电量浮动以平衡供需偏差：
    grid_actual(t) = ele_demand_actual(t) - local_supply_plan(t)
  - 如果 grid_actual(t) < 0：弃电（或卖回电网，收益较低）
  - 如果 grid_actual(t) > 0：额外购电（按实时电价）

计算：
  - 实际运行成本 = Σ(gas_cost + grid_cost + penalty)
  - penalty = 弃电量 × 弃电惩罚系数 + 供需失衡量 × 失衡惩罚系数
```

### 5.4 评价指标


| 指标        | 公式                                                | 含义          |
| --------- | ------------------------------------------------- | ----------- |
| 运行成本增量 ΔC | (C_S2 - C_S1) / C_S1 × 100%                       | 预测误差导致的额外成本 |
| 峰值购电增量 ΔP | (P_peak_S2 - P_peak_S1) / P_peak_S1 × 100%        | 电网冲击恶化      |
| 供需失衡时数    | Σ{t: |grid_actual(t) - grid_plan(t)| > threshold} | 调度可靠性       |
| 弃能增量      | curtail_S2 - curtail_S1                           | 可再生消纳恶化     |


### 5.5 预期结果


| 指标   | EQD方案退化 | Std方案退化  | EQD+CB方案退化 | 解释           |
| ---- | ------- | -------- | ---------- | ------------ |
| ΔC   | 小(2-5%) | 大(5-10%) | 最小(1-3%)   | EQD配了更多电储能缓冲 |
| ΔP   | 小       | 大        | 最小         | EQD电网交互本身更平滑 |
| 失衡时数 | 少       | 多        | 最少         | 卡诺电池提供额外缓冲   |


**核心论点**：

1. 㶲加权规划方案不仅在完美信息下优于等权重方案（第2/3章已证明），
  在面对预测不确定性时也展现出更强的鲁棒性。
2. 原因：㶲加权方法优先保障电平衡（λ_e=1.0 >> λ_h=0.131），
  配置了更多电储能和灵活性资源，这些资源恰好也是应对预测误差的缓冲。
3. 卡诺电池的电热耦合特性提供了额外的跨品位缓冲能力。
