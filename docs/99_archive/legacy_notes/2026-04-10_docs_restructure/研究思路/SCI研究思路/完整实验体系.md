# 完整实验体系设计

## 总览：论文要讲的故事

```
创新点1（方法创新）：基于㶲加权的源荷匹配度指标
  → 实验1 证明方法有效（德国案例，5种方法对比）
  → 实验2 证明方法普适（松山湖案例，std vs euclidean）
  → 实验5 证明匹配度有实际价值（后验运行指标分析）
  → 实验6 证明匹配度带来韧性（极端场景验证）

创新点2（模型创新）：含卡诺电池的IES容量优化
  → 实验3 证明卡诺电池有价值（松山湖，有/无卡诺电池）
  → 实验4 证明两个创新点可以叠加（松山湖+卡诺，std vs euclidean）
```

## 实验1：方法对比（创新点1的主实验）

**回答的问题：** 你的匹配度指标比其他4种方法好在哪？

**设置：**
- 案例：德国（热主导）
- 方法：5种全跑（economic_only / std / euclidean / pearson / ssr）
- 参数：nind=50, maxgen=100
- λ：卡诺㶲系数 [1.000, 0.131, 0.064]

**输出：**
- 5条Pareto前沿画在同一张图上
- 每种方法的折中解（TOPSIS）的设备配置表

**怎么分析：**
1. Pareto前沿对比图：euclidean的前沿是否更靠近原点（同成本下匹配度更低）
2. 设备配置表：euclidean推荐的配置是否更合理（比如不会过度配CHP或过度配储能）
3. 各方法的缺陷是否如预期：
   - economic_only：成本最低但匹配度最差
   - std：不区分能源品位，电热冷等权对待
   - pearson：只看趋势不看大小，可能配置容量偏小
   - ssr：只看总量不看时序分布

**对应 run.py：** `--exp 1`

## 实验2：普适性验证（创新点1的补充）

**回答的问题：** 换一个完全不同的案例（冷主导 vs 热主导），方法还有效吗？

**设置：**
- 案例：松山湖（冷主导，亚热带气候）
- 方法：std + euclidean（只需对比核心方法）
- λ：卡诺㶲系数 [1.000, 0.099, 0.071]（自动适应松山湖温度参数）

**输出：**
- 2条Pareto前沿对比图
- 设备配置对比表

**怎么分析：**
1. 在冷主导案例下，euclidean是否仍然优于std
2. λ自动变化（松山湖λ_c > 德国λ_c）是否合理地引导了优化方向
3. 松山湖的配置特征：EC/AC/CS应该更大（冷主导），PWT=0（无风资源）

**对应 run.py：** `--exp 2`

## 实验3：卡诺电池价值（创新点2）

**回答的问题：** 加入卡诺电池后，系统的匹配度和经济性如何变化？

**设置：**
- 案例：松山湖
- 方法：euclidean（用你的最优方法）
- 对比：无卡诺电池 vs 有卡诺电池

**输出：**
- 2条Pareto前沿对比图（有/无卡诺电池）
- 卡诺电池的最优容量配置
- 卡诺电池余热回收对热匹配度的贡献

**怎么分析：**
1. 加入卡诺电池后Pareto前沿是否整体左移/下移（成本降低或匹配度改善）
2. 卡诺电池在系统中扮演什么角色（储电？余热回收？削峰填谷？）
3. 卡诺电池的经济性拐点：多大容量时边际收益开始递减

**对应 run.py：** `--exp 3`

## 实验5：后验运行指标分析（证明匹配度的实际价值）⭐ 新增

**回答的问题：** 匹配度更好的方案，在实际运行中到底好在哪？

这是回答审稿人"匹配度有什么用"的关键实验。

**设置：**
- 输入：实验1和实验2的Pareto前沿结果
- 从每种方法的Pareto前沿上，在3个成本水平（低/中/高）各取1个方案
- 用完整8760小时数据重新跑每个方案的调度模型

**具体做法：**

```
Step 1: 确定成本水平
  - 取所有方法Pareto解的成本范围交集
  - 在交集内选3个成本水平：25%分位、50%分位、75%分位

Step 2: 在每个成本水平下，从每种方法的Pareto前沿上找最近的解
  - 比如成本水平=50万€，从std前沿上找成本最接近50万的解，
    从euclidean前沿上也找一个

Step 3: 对每个选出的解（设备容量已固定），跑完整8760小时调度
  - 不是用14个典型日，而是逐日跑365天×24小时
  - 记录每小时的：购电量、弃电量、弃热量、弃冷量、各设备出力

Step 4: 计算运行指标
```

**输出表格（论文核心Table之一）：**

| 成本水平 | 方法 | 年化成本 | 峰值购电 | 年购电量 | 弃光率 | 自给自足率 | 电网波动 |
|---------|------|---------|---------|---------|-------|-----------|---------|
| 50%分位 | economic_only | 45万€ | 2500kW | 8000MWh | 15% | 65% | 800kW |
| 50%分位 | std | 48万€ | 2200kW | 7500MWh | 12% | 70% | 700kW |
| 50%分位 | **euclidean** | 48万€ | **1800kW** | **6800MWh** | **8%** | **75%** | **500kW** |

（以上数字是预期方向的示意，实际数值需要跑出来）

**预期结论：**
在相同成本水平下，euclidean方法得到的方案：
- 峰值购电更低 → 电网友好，减少配电网投资
- 弃光率更低 → 可再生能源利用率高，政策合规
- 自给自足率更高 → 对外依赖低，韧性强
- 电网交互波动更小 → 调度友好

**这就回答了"为什么不直接优化这些指标"：**
- 这些指标是离散的、非光滑的（涉及max/min操作），不适合做GA优化目标
- 匹配度是连续、可微、多维耦合的代理指标，适合引导GA搜索
- 用匹配度优化，用运行指标评价 = 用loss训练，用accuracy评价

**对应代码：** 需要新增 `scripts/post_analysis.py`

## 实验6：极端场景韧性验证（加分项）⭐ 新增

**回答的问题：** 匹配度高的方案在极端天气下是否更抗风险？

**设置：**
- 输入：实验5中选出的方案（容量已固定）
- 从8760小时数据中人为选取极端日

**极端日选取方法：**

```python
# 从真实数据中找极端日，不是人造的
# 德国案例
extreme_days = {
    "夏季极热日": 找cool_load最大的那天,      # 冷负荷峰值
    "冬季极冷日": 找heat_load最大的那天,      # 热负荷峰值
    "低光照日":   找solarRadiation最小的工作日, # 光伏几乎为零
    "高负荷日":   找ele_load最大的那天,        # 电负荷峰值
}
```

**对每个极端日，比较不同方案的表现：**

| 极端场景 | 方案来源 | 日运行成本 | 最大购电 | 供能缺口 | 弃能量 |
|---------|---------|-----------|---------|---------|-------|
| 夏季极热 | economic_only | 高 | 大 | 有 | 少 |
| 夏季极热 | std | 中 | 中 | 小 | 中 |
| 夏季极热 | **euclidean** | 中 | **小** | **无** | **少** |

**预期结论：**
- 匹配度高的方案在极端日的额外成本增量更小
- 匹配度高的方案不会出现供能缺口
- 这证明了：正常场景下多花一点钱（Pareto前沿上成本稍高），换来极端场景下的安全保障

**对应代码：** 可以集成在 `scripts/post_analysis.py` 中

```
  ├── 实验2: 松山湖×std+euclidean   ~1小时

Phase 3: 出图出表（几分钟）
  ├── Pareto前沿对比图（实验1-4各一张）

```
