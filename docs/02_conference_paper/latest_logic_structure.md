# 会议论文：最新逻辑结构

更新时间：2026-04-10

## 1. 定位

会议论文的任务不是“把整个研究都讲一遍”，而是**把规划层里的一个核心概念讲清楚并做硬证据验证**。

当前会议稿的最简主张是：

> 源荷匹配不能只看各能流各自的波动，还要看每个时刻电-热-冷联合失配的向量长度；IEMI 比 Std 更能体现这种时刻级耦合，因此能导向更好的规划方案与运行表现。

## 2. 会议稿必须回答的三个问题

1. 为什么 `std` 这类“各能流先分开统计再相加”的指标不够？
2. 为什么“时刻级三维净失配向量 + 欧氏范数”是更合理的规划层表达？
3. 这种表达有没有带来可观测的规划结果差异和年度运行收益？

## 3. 会议稿的证据边界

### 必须保留的证据

- 德国案例的概念验证。
- `economic_only / std / euclidean(IEMI)` 三组方法对比。
- Pareto 前沿差异。
- 在相近预算下的 8760h 后验运行指标对比。

### 可以轻提但不能展开成主线的内容

- 㶲权重 / EQD：只作为“期刊扩展”的方向点到为止。
- 卡诺电池：最多放在 discussion 或 future work，不要塞进会议正文主体。
- 负荷预测、日前调度、鲁棒性：不属于会议稿范围。

## 4. 建议文章结构

1. `Introduction`：问题背景、现有匹配度写法的不足、本文要验证的核心概念。
2. `System modeling`：CCHP 拓扑、典型日、双目标规划的最小必要描述。
3. `Matching index`：IEMI 的定义、几何意义、与 Std 的关键差异。
4. `Optimization`：双目标框架与算法设置。
5. `Case studies`：德国案例、参数、方法集合。
6. `Results`：Pareto 对比 + 预算对齐后验运行指标。
7. `Conclusions`：明确会议稿只完成 concept validation，为 SCI 扩展留出口。

## 5. 术语约束

- 会议稿主术语：**IEMI**。
- 不把会议稿写成“EQD + 卡诺电池 + 完整 SCI 故事”的压缩版。
- 如果必须提到 EQD，只能说明：IEMI 是这条方法主线的会议版表达，SCI 稿会进一步补上㶲加权与设备层扩展。
