# 最新研究架构

更新时间：2026-04-10

## 1. 需要同时记住的两套“三层”

### 1.1 论文载体三层

- **会议论文**：先把一个核心点讲清、讲硬，定位是 concept validation。
- **SCI 论文**：把核心创新扩展成完整论文故事，实验链条更完整，证据更强。
- **硕士论文**：把前两者纳入一个更大的研究闭环，强调章节递进与整体贡献。

### 1.2 研究内容三层

- **规划层**：㶲加权匹配度 / IEMI / EQD，回答“规划阶段怎样量化源荷匹配”。
- **设备层**：卡诺电池集成与配置价值，回答“新型电热耦合储能值不值得配、该怎么配”。
- **运行层**：负荷预测、日前调度、鲁棒性验证，回答“规划方案在预测误差和运行不确定性下是否仍然站得住”。

## 2. 当前统一主线

当前整个研究不再按“零散方法点子”组织，而是按“**规划层方法 -> 设备层扩展 -> 运行层闭环验证**”组织。

一句话主线是：

> 在分布式电热综合能源系统中，先提出能够体现能源品质差异和多能耦合失衡的规划层匹配度量，再验证其对新型耦合储能配置的识别能力，最后在运行层检验其面对预测误差时的鲁棒性。

## 3. 三个载体分别讲什么

### 3.1 会议论文：只讲规划层的一个核心点

会议稿不再追求把整套研究一次讲完，而是聚焦“**时刻级三能流联合失配要按向量方式看，而不是各能流各算各的再相加**”这个核心概念。

- 会议稿当前主术语是 **IEMI**，强调 instant-level、vector-level、Euclidean 三个关键词。
- 代码层面的目标函数仍对应 `euclidean` 这条方法主线。
- 㶲加权和 EQD 只作为向 SCI 扩展的前导关系，不在会议稿里展开成长篇第二定律推导。

### 3.2 SCI 论文：规划层完整展开 + 设备层价值验证

SCI 稿承担“完整论文故事”的角色，核心是两步：

1. 把会议稿中的 IEMI 几何思想升级为有热力学依据的 **EQD（㶲加权欧氏距离）**。
2. 继续向前走到 **卡诺电池**，验证该匹配度指标是否能够识别电热耦合储能的配置价值。

因此，SCI 的核心层次是：

- 规划层：EQD 对比 Economic / Std / Pearson / SSR，建立方法优势。
- 设备层：无卡诺 vs 有卡诺、Std vs EQD with Carnot，建立设备价值与方法协同。
- 运行层：只保留后验运行指标、敏感性和韧性作为“强证据链”，不把完整预测-调度章节塞进期刊正文主线。

### 3.3 硕士论文：规划层 + 设备层 + 运行层完整闭环

硕士论文是唯一需要把三层全部讲完的载体。

- **第 2 章**承接规划层：从 IEMI / EQD 的方法问题出发，建立多目标规划主线。
- **第 3 章**承接设备层：围绕卡诺电池，把“方法能否识别耦合设备价值”讲透。
- **第 4 章**承接运行层：引入负荷预测、日前调度、鲁棒性验证，把“规划好不好”落到“运行时扛不扛得住”上。

## 4. 术语对应关系

为了避免未来助手混淆，统一按下面理解：

- `euclidean`：代码里的方法名，是同一条主线。
- **IEMI**：会议论文中的轻量表达，突出“时刻级向量失配”。
- **EQD**：SCI / 硕士论文中的正式表达，突出“㶲加权欧氏距离”的物理含义。

换句话说，**会议论文讲的是这条方法主线的几何概念验证版，SCI / 硕士论文讲的是它的热力学完整展开版**。

## 5. 当前研究状态

- **规划层**：已在当前代码与实验体系中落地，是现阶段最成熟的主线。
- **设备层**：卡诺电池已进入模型和实验设计，属于 SCI 与硕士论文的第二层核心贡献。
- **运行层**：逻辑边界已明确，仓库里已有 `scripts/resilience_test.py` 等支撑脚本，但完整的“负荷预测 + 日前调度 + 误差鲁棒性”模块仍属于硕士论文后续重点建设内容。
