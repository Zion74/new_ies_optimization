## 2. Phase 2 不可行解污染（核心发现）

### 2.1 污染规模

扫描两批次所有 `Pareto_*.csv`（阈值 `Economic_Cost > 1e8` 视为 GA 惩罚值）：

| 场景 | 批 1 `Euclidean` 不可行占比 | 批 2 `Euclidean` 不可行占比 |
|---|---:|---:|
| `exp1` 德国 base | 0 / 80 | 0 / 80 |
| `exp2a` 德国 base | 0 / 80 | 0 / 80 |
| `exp2b` 德国 + Carnot | 1 / 80 (1.2%) | 0 / 80 |
| `exp3` 松山湖 base | **43 / 80 (53.8%)** | **44 / 80 (55.0%)** |
| `exp4` 松山湖 + Carnot | **31 / 80 (38.8%)** | **18 / 80 (22.5%)** |

`economic_only` / `std` / `pearson` / `ssr` 在两批次的五组实验中**全部 0 污染**。

### 2.2 根因

`cchp_gaproblem.sub_aim_func_cchp()` 对 OEMOF 求解失败的个体赋予 1e9~1e12 级别的惩罚值，NSGA-II 在非支配排序时不会把它们踢出 Pareto（它们在经济维度被所有可行解支配，但在匹配度维度有时是极小值，导致它们作为"匹配度最优端点"保留下来）。

**为什么只有 `euclidean` 在松山湖受灾**：
- `euclidean` 目标推 GA 向"零能流净交换"极端方向搜，这种极端配置（低 GT + 低 HP 等）在松山湖的冷主导负荷下高频触发 OEMOF infeasibility
- 德国案例的冷热电更均衡，这类极端配置仍大多可行
- `std` 目标是波动率，极端方向与匹配度最优方向不重合，不容易触发 infeasibility

### 2.3 对下游的实际污染

`scripts/post_analysis.py::select_solutions_at_cost_levels()` 用所有方法 Pareto 并集的 `cost_min/cost_max` 线性分档。污染解把 `cost_max` 推到 1e11~1e12，导致 `absolute` Level 2/3 全部落在不可行区。结果表现：

- `exp2b` 批 1 Level 3 `euclidean` `annual_cost` = **7.95 × 10¹⁰**
- `exp4` 批 1 Level 2 `euclidean` `annual_cost` = **3.40 × 10¹¹**
- `exp4` 批 1 Level 3 `euclidean` `annual_cost` = **2.23 × 10¹²**
- `exp4` 批 2 Level 2 `euclidean` `annual_cost` = **7.51 × 10¹¹**
- `exp3` 批 1/2 Level 3 `euclidean` `annual_cost` = **7.95 × 10¹⁰ ~ 4.80 × 10¹¹**

所有超过 1e9 的行都被 `phase2_sentinel_filter.py` 标记为不可行，从 `_clean.csv` 剥离到 `post_analysis_infeasible.csv`。

### 2.4 为什么阈值选 1e8

| 边界 | 值 |
|---|---:|
| 德国业务成本上界 | ~3 × 10⁶ |
| 松山湖业务成本上界 | ~3 × 10⁶ |
| 合法 Pareto cost_max（松山湖 `euclidean`，过滤后） | ~2.1 × 10⁶ |
| 污染解下界（所有样本中最小） | ~4 × 10¹¹ |

业务上界到污染下界相差 **5 个数量级**。取中间 1e8（比业务最大值高 33 倍、比污染最小值低 4000 倍），不可能误伤合法解，也不可能漏掉惩罚值。

## 4. 对实验设计的建议（还未落地）

### 4.1 创新点 2（Carnot 价值）叙事重构

当前用"`exp2b` `euclidean` 最低成本"作为 Carnot 价值证据，因为端点的 9.5% 跨批差直接站不住。建议改为二选一：

1. **Pareto 超体积（hypervolume）**：对 `exp2a`（无 Carnot）和 `exp2b`（有 Carnot）的 `euclidean` Pareto 前沿分别计算 hypervolume（参考点取 `max(cost) × 1.1, max(matching) × 1.1`），比 `ΔHV = HV(with) - HV(without)`。这个指标对端点不敏感。
2. **膝盖点（knee point）**：对两条 Pareto 用 L-method 或 Kneedle 识别拐点，比拐点的经济成本与匹配度。拐点在 Pareto 中段，GA 随机性最小。

### 4.2 补第 3 次独立运行（可选但强烈建议）

当前两次已经能支持大部分定性结论，但 `exp2b` `euclidean` 这种边缘情况需要第 3 次以上给出均值 ± 标准差。建议：

- 复用 `scripts/openbayes_run_experiments.sh`，加 `IES_RUN_TAG=post_rebuild_batch3` 跑第三次
- 三次后用 `scripts/compare_parameter_scales.py` 或类似脚本聚合成 mean ± std 表

### 4.3 不用动

- 参数规模（80/150 已到性价比拐点，进一步提到 100/200 对 `exp2b` 端点稳定性改善有限，根因是端点敏感性而非收敛性）
- 实验编号、案例、方法列表（架构合理，只是度量方式要换）
